Configuring iFolder Preferences for the Client

  1. Right-click the iFolder Services icon in the notification area, then select Preferences to open the iFolder Preferences dialog box to the General tab.

  2. Specify your preferences for the following:

    Parameter

    Description

    Start iFolder when logging in to the desktop (Available only in windows client)

    Select this check box to launch iFolder whenever you log in with the local username where you set up the iFolder account. For those accounts where you have enabled Remember My Password, you will not be prompted to specify your password when you log in.

    Deselect this check box to start iFolder manually.

    Display confirmation dialog on successful creation of iFolder

    Select this check box to receive a confirmation message each time you create an iFolder, in addition to the normal visual cues such as the iFolder appearing in the list of iFolders.

    Deselect this check box to stop receiving the confirmation message.

    Hide iFolder Tray Icon (Available only in windows client)

    Select this check box to hide the iFolder icon displayed in system tray.

    Deselect this check box to display the iFolder icon in the system tray.

    WARNING:If you select this check box, ensure that the Hide iFolder main window at startup check box is deselected. Otherwise, the visual cues that indicate a running iFolder application will be unavailable.

    Hide iFolder main window at startup

    Select this check box if you want the iFolder window to remain hidden on startup.

    Deselect this check box to display the iFolder window on startup.

    Display synchronization logs

    Select this check box if you want to display synchronization logs on startup.

    Deselect this check box if you want to hide synchronization logs on startup.

    When quota policy is violated

    Select this check box to display a pop-up message in the notification area whenever a disk quota policy is violated.

    Deselect this check box to disable this notification.

    When file size policy is violated

    Select this check box to display a pop-up message in the notification area whenever a file size policy is violated.

    Deselect this check box to disable this notification.

    When file exclusion policy is violated

    Select this check box to display a pop-up message in the notification area whenever a file exclusion policy is violated.

    Deselect this check box to disable this notification.

    When disk is full

    Select this check box to display a pop-up message in the notification area whenever disk is full in the local system.

    Deselect this check box to disable this notification.

    When required permissions are unavailable

    Select this check box to display a pop-up message in the notification area whenever the required permissions to perform an operation on a file or folder is unavailable. For example, you will be unable to download a file if you have only read rights.

    Deselect this check box to disable this notification.

    When file path exceeds optimal limit

    Select this check box to display a pop-up message in the notification area whenever a file path exceeds the system-defined maximum length. For example, on Windows-based platforms, the number of characters in a path must be less than 248.

    Deselect this check box to disable this notification.

    When iFolders are shared

    Select this check box to display a pop-up message in the notification area whenever you receive a new invitation to share an iFolder.

    Deselect this check box to disable this notification.

    When conflicts arise

    Select this check box to display a pop-up message in the notification area whenever a conflict occurs on synchronizing files.

    Deselect this check box to disable the pop-up messages for synchronization conflicts.

    Whenever conflicts occur, the iFolder Error icon is displayed on the iFolder.

    Synchronization

    Automatically Synchronize iFolders: Select this check box to enable synchronization for all iFolders on this computer under your current local login or logon identity. Deselect the check box to disable synchronization.

    Synchronize iFolders Every: Specify the minimum interval to use for synchronizing iFolders on this computer under your current local login or logon identity. Specify the value and units. The default is 5 minutes if this option is enabled. The minimum interval can be set to 5 seconds.

    If you use the iFolder client with Novell iFolder 3.7 or later server, the effective minimum synchronization interval is always the largest value of the following settings:

    • The system policy default of zero (0), unless there is a user policy set. If a user policy is set, the user policy overrides the system policy, whether the user policy is larger or smaller in value.

    • The local machine policy, or the setting on the client machine synchronizing with the server.

    • The iFolder (collection) policy.

  3. Click Close to close the iFolder Preferences dialog box.
